# PAIKS — Personal AI Knowledge System

PAIKS is a locally-hosted AI document assistant that connects to your Google Drive, indexes your documents with semantic embeddings, and lets you ask natural-language questions answered by a local LLM (Ollama). No cloud AI APIs are used — everything runs on your own machine.

## 1. Project Overview

PAIKS solves a common problem: your knowledge is scattered across dozens of Google Docs and text files, and you cannot search *inside* them — only by filename.

With PAIKS you can:

- **Connect** your Google Drive account (read-only OAuth)
- **Sync** a list of your documents (Google Docs, .docx, .txt, .csv)
- **Index** the full text of those documents into a local vector database
- **Ask** natural-language questions and get answers grounded in your actual document content, generated by a local LLM
- **Browse** and filter your Drive files in a clean UI

All data (tokens, embeddings, cache) stays on your local machine. No third-party AI API is used.

## 2. Architecture

PAIKS runs as two separate Python servers that work together:

```
Browser
  │
  ├──► Django (port 8000) ─── serves HTML/CSS/JS UI only
  │
  └──► Flask  (port 5001) ─── all API logic
         │
         ├── Google Drive API (OAuth2, read-only)
         ├── ChromaDB (local vector store at ./chroma_db/)
         ├── ONNX Embeddings (all-MiniLM-L6-v2, bundled with chromadb)
         └── Ollama (local LLM at http://localhost:11434)

XAMPP / Apache (port 80)
  └── /PAIKS/oauth2callback  ←── Google redirects here after OAuth consent
        │  (oauth2callback.php)
        └── forwards ?code= to Flask /auth/callback
```

**Why two servers?**
Django is excellent at serving templated HTML. Flask is lightweight and perfect for a REST API gateway. Keeping them separate makes each part independently testable and replaceable.

**Why XAMPP?**
Google OAuth requires a registered redirect URI. During local development, the redirect URI is `http://localhost/PAIKS/oauth2callback`, which XAMPP serves via Apache. A small PHP bridge (`oauth2callback.php`) forwards the authorization code to the Flask server.

## 5. One-Time Setup

### 5.1 Google Cloud Console — OAuth Credentials

You need a Google Cloud project with the Drive API enabled and OAuth credentials downloaded.

1. Go to [https://console.cloud.google.com/](https://console.cloud.google.com/)
2. Create a new project (or select an existing one)
3. Go to **APIs & Services → Library** → search for **Google Drive API** → click **Enable**
4. Go to **APIs & Services → OAuth consent screen**
   - User type: **External**
   - Fill in app name (e.g. `PAIKS`), your email, and save
   - Add your Google account as a **Test user**
5. Go to **APIs & Services → Credentials → Create Credentials → OAuth client ID**
   - Application type: **Web application**
   - Name: `PAIKS Local`
   - Authorized redirect URIs — add **both**:
     ```
     http://localhost/PAIKS/oauth2callback
     http://localhost/PAIKS/oauth2callback/
     ```
6. Click **Create**, then **Download JSON**
7. Rename the downloaded file to `credentials.json` and place it in the project root:
   ```
   C:\xampp\htdocs\PAIKS\credentials.json
   ```

---

### 5.2 XAMPP (Apache) — OAuth Callback Bridge

XAMPP's Apache serves `http://localhost/PAIKS/` which is required for the Google OAuth redirect.

1. **Install XAMPP** from [https://www.apachefriends.org/](https://www.apachefriends.org/) if not already installed
2. Open **XAMPP Control Panel** and start **Apache**
3. Verify the project is accessible at `http://localhost/PAIKS/` in your browser
4. The OAuth callback files are already in place:
   - `C:\xampp\htdocs\PAIKS\oauth2callback.php`
   - `C:\xampp\htdocs\PAIKS\oauth2callback\index.php`

> **Note:** Both files do the same thing — forward `?code=` from Google to Flask's `/auth/callback`. Google sometimes redirects to the file or the directory, so both are provided.

### 5.4 Ollama (Local LLM)

Ollama runs a local LLM that generates answers from retrieved document context.

1. **Download and install Ollama** from [https://ollama.com/download](https://ollama.com/download)
   - It installs as a Windows service and starts automatically on port `11434`

2. **Pull a model** — open any terminal and run:

   ```bash
   ollama pull llama3.2
   ```

   Other good options (smaller = faster):

   | Model | Size | Speed | Quality |
   |---|---|---|---|
   | `llama3.2` | ~2 GB | Fast | Very good |
   | `mistral` | ~4 GB | Medium | Excellent |
   | `gemma2:2b` | ~1.5 GB | Very fast | Good |
   | `phi4` | ~9 GB | Slow | Best |

3. Verify Ollama is running:
   ```
   http://localhost:11434
   ```
   You should see: `Ollama is running`

## 7. Using PAIKS

### Step 1 — Connect Google Drive

1. Open `http://127.0.0.1:8000/`
2. Click **Connect Drive**
3. You will be redirected to Google's OAuth consent screen
4. Sign in and grant read-only access
5. You will be redirected back to the home page with a "Connected" status

### Step 2 — (Optional) Choose a Folder

By default, PAIKS syncs your entire Drive. To limit it to a specific folder:

1. Go to **Dashboard** (`/dashboard/`)
2. Click **Choose Folder** and select a folder from the picker
3. Only files inside that folder will be synced and searched

### Step 3 — Sync Your Files

1. On the Dashboard, click **Sync Now**
2. PAIKS will fetch all your matching documents (Google Docs, .docx, .txt, .csv) from Drive
3. The file list is saved to `drive_cache.json`

### Step 4 — Index Documents for RAG

1. Go to **Files** (`/files/`)
2. Click **⚡ Index Docs**
3. PAIKS will:
   - Download the full text content of each synced file
   - Split each document into overlapping chunks (~600 characters)
   - Embed each chunk using `all-MiniLM-L6-v2` (ONNX, runs locally)
   - Store the embeddings in ChromaDB (`./chroma_db/`)
4. The status badge will show **✓ N chunks indexed**

> This may take a minute or two depending on how many documents you have. It only needs to be done once, or after adding new documents (re-index with the same button).

### Step 5 — Configure Local LLM

At the bottom of the Files page, in the **LLM config bar**:

1. Select **Ollama** as the provider
2. Leave the URL as `http://localhost:11434`
3. Select your model from the dropdown (e.g. `llama3.2`)
4. Click **Apply**
5. The status dot should turn **green**

### Step 6 — Ask Questions

In the **Smart RAG Search** panel on the Files page:

1. Type a natural-language question, e.g.:
   - *"What are the Q3 revenue figures?"*
   - *"Summarise the project requirements document"*
   - *"Who is responsible for the marketing campaign?"*
2. Press **Enter** or click **Ask**
3. PAIKS will:
   - Embed your question and find the most semantically similar chunks
   - Send the top chunks as context to your local Ollama model
   - Display the **AI Answer** card at the top
   - Display the **source documents** with relevance scores and text snippets below

## 8. Flask API Reference

The Flask server runs at `http://127.0.0.1:5001`.

### Auth

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/health` | Health check — returns `{"status": "ok"}` |
| `GET` | `/auth/status` | Returns `{"authenticated": true/false}` |
| `GET` | `/auth/url` | Returns the Google OAuth redirect URL |
| `GET` | `/auth/callback` | Handles the OAuth code exchange (called by Apache bridge) |
| `POST` | `/auth/disconnect` | Deletes token and cache, logs out |

### Google Drive

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/drive/folders` | List all Drive folders for the picker |
| `POST` | `/drive/set-folder` | Save folder scope `{"folder_id": "...", "folder_name": "..."}` or `{}` to clear |
| `GET` | `/drive/folder-config` | Return current folder scope |
| `GET` | `/drive/files` | Live paginated file listing. Params: `?q=`, `?pageToken=`, `?pageSize=` |
| `POST` | `/drive/sync` | Full sync — fetches all matching files, writes `drive_cache.json` |
| `GET` | `/drive/stats` | Returns total count, last sync time, file type breakdown |

### Search

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/search` | Name-based keyword search. Body: `{"query": "..."}` |

### RAG

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/rag/status` | Returns `{"indexed": bool, "total_chunks": N, "ingest_running": bool}` |
| `POST` | `/rag/ingest` | Download all synced files, extract text, chunk, embed, store in ChromaDB |
| `POST` | `/rag/search` | Semantic search + LLM answer. Body: `{"query": "..."}`. Returns `answer`, `results`, `source` |
| `GET` | `/rag/llm/status` | Ping local LLM, returns `{"reachable": bool, "available_models": [...]}` |
| `POST` | `/rag/llm/config` | Save LLM settings. Body: `{"base_url": "...", "model": "...", "provider": "ollama"}` |

## 9. How the RAG Pipeline Works

```
User types a question
        │
        ▼
1. EMBED QUERY
   ChromaDB's DefaultEmbeddingFunction encodes the question
   into a 384-dimensional vector using all-MiniLM-L6-v2 (ONNX)
        │
        ▼
2. RETRIEVE (ChromaDB cosine similarity search)
   Top 15 most similar document chunks are retrieved
   Deduplicated to best chunk per source file → shown as sources
   Top 5 raw chunks → sent to LLM as context
        │
        ▼
3. GENERATE (Ollama — local LLM)
   Prompt:
     "Answer using ONLY the documents below...
      [Source: file1.docx] chunk text...
      [Source: file2.txt] chunk text...
      QUESTION: {user query}
      ANSWER:"
        │
        ▼
4. RESPOND
   {
     "answer": "The Q3 revenue was $2.4M based on...",
     "answer_model": "llama3.2",
     "results": [ {name, snippet, score, webViewLink}, ... ],
     "source": "semantic"
   }
```

**Chunking strategy:** Documents are split into 600-character overlapping windows with 120-character overlap. This keeps each chunk small enough to be semantically focused while preserving sentence continuity across chunk boundaries.

**Fallback behaviour:** If ChromaDB has no data (not indexed yet), the search falls back to live Drive API name-based search. If Ollama is unreachable, the source document list is still returned without a generated answer.

## 11. Troubleshooting

### "Not connected" after clicking Connect Drive

- Make sure XAMPP Apache is running on port 80
- Check that `credentials.json` is in the project root
- Verify the redirect URI in Google Cloud Console exactly matches `http://localhost/PAIKS/oauth2callback`
- Check Flask terminal for token exchange errors

### Flask API not reachable

- Make sure `flask_api.py` is running: `.venv\Scripts\python flask_api.py`
- Confirm it's on port 5001: check for `Running on http://127.0.0.1:5001`
- Check Windows Firewall isn't blocking localhost connections

### "No synced files. Run Sync first."

- Go to Dashboard and click **Sync Now** before trying to index
- If no files appear, check that your Drive has .docx, .txt, Google Docs, or .csv files

### Local LLM status dot is red

- Ensure Ollama is installed and running: open `http://localhost:11434` in a browser
- If Ollama is installed but not running: open a terminal and run `ollama serve`
- Make sure you have at least one model pulled: `ollama pull llama3.2`
- In the LLM config bar, set URL to `http://localhost:11434`, select a model, click **Apply**

### RAG search returns no answer (only sources)

- The LLM config bar must show a green dot before answers are generated
- If the dot is red, Ollama is not reachable (see above)
- Check the Flask terminal for `Local LLM call failed:` error messages

### `WinError 1114` DLL error with torch

- This is a Visual C++ Runtime issue. PAIKS does **not** use PyTorch — it uses ChromaDB's built-in ONNX embeddings
- If you see this error, ensure `sentence-transformers` is not installed: `.venv\Scripts\pip uninstall sentence-transformers`

### Re-indexing after adding new documents

1. Add new files to your Google Drive (in the synced folder if one is set)
2. Go to Dashboard → **Sync Now** (updates `drive_cache.json`)
3. Go to Files → **⚡ Index Docs** (re-indexes — existing chunks are upserted, not duplicated)

## Supported Document Types

| Format | MIME Type | How text is extracted |
|---|---|---|
| Google Docs | `application/vnd.google-apps.document` | Drive export API → plain text |
| Word (.docx) | `application/vnd.openxmlformats-officedocument...` | Download + python-docx parser |
| Word (.doc) | `application/msword` | Download + python-docx parser |
| Plain text (.txt) | `text/plain` | Direct download |
| CSV (.csv) | `text/csv` | Direct download |

PDFs, images, spreadsheets, and presentations are not indexed (Drive's export API does not support plain-text extraction for these types in this configuration).
